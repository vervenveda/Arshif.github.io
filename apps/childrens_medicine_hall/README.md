# ARSHIF Children’s Medicine Resource & Information Portal

A safety-first, self-contained educational portal for parents, caregivers, teachers, and older students who want a clearer way to organize child-health questions, medicine-safety information, emergency warning signs, family notes, and trusted pediatric source pathways.

This project is part of the ARSHIF educational hall ecosystem and is designed for static hosting on GitHub Pages or any simple web server.

> **Important medical boundary:** This portal is educational only. It does not diagnose, prescribe, calculate doses, replace a pediatrician, replace Poison Control, replace emergency services, or replace local medical guidance.

## Core Safety Principles

This portal is built around a conservative child-safety frame:

1. **Emergency symptoms come first.** Severe trouble breathing, collapse, seizure, inability to wake, blue/gray lips, severe allergic reaction, major injury, or life-threatening concern should trigger local emergency services.
2. **Poison concerns need expert help.** In the United States, families should contact Poison Control for suspected poisoning or overdose unless emergency symptoms are present, in which case emergency services come first.
3. **No dosing engine is included.** Children’s dosing depends on age, weight, medicine concentration, medical conditions, allergies, interactions, and clinician guidance.
4. **Medicines, supplements, vitamins, gummies, essential oils, and remedies should be stored safely.** Natural does not automatically mean safe for children.
5. **Complementary or traditional care should never delay emergency or proven medical care.**
6. **Parent/caregiver notes are private records and should not be uploaded to GitHub.**

## GitHub Pages Setup

To publish this as a simple GitHub Pages site:

1. Create or open your GitHub repository.
2. Upload the HTML file.
3. Rename the main portal file to `index.html` if you want it to load as the homepage.
4. Add this `README.md` file to the repository root.
5. Go to repository **Settings → Pages**.
6. Choose the branch and folder you want to publish from.
7. Save and wait for GitHub Pages to provide the public URL.

## Privacy Warning

The portal uses browser storage for notes, logs, report drafts, favorites, quiz progress, and Family Vault information.

That means:

- Data is stored on the current device/browser.
- Data does not automatically sync between computers.
- Data may be lost if browser storage is cleared.
- Shared or public computers are not recommended for private family records.
- Exported Family Vault JSON files may contain sensitive information.

### Do not upload private records to GitHub

Do **not** upload any of the following to a public repository:

- Exported Family Vault JSON files
- Medicine logs
- Child profiles
- Emergency contacts
- Pediatrician notes
- Allergy action notes
- Private reports
- Any document containing a child’s personal or medical information

Only upload the static portal HTML file and public project files such as `README.md`, `LICENSE.md`, and notices.

## Suggested Backup Routine

For families using the portal seriously:

- **Weekly:** Export the Family Vault JSON.
- **Monthly:** Print or save the family safety report.
- **Before appointments:** Export or print relevant notes and questions.
- **Before switching computers:** Export the Family Vault and import it on the new device.
- **Before clearing browser data:** Export the Family Vault.

Keep backups in a private, secure place controlled by the parent or guardian.

## Source Philosophy

The portal points families toward durable, trusted source pathways, including pediatric and public-health references such as:

- American Academy of Pediatrics / HealthyChildren
- Poison Control
- FDA consumer guidance
- CDC child health and vaccine resources
- MedlinePlus
- NIH Office of Dietary Supplements
- NCCIH children and complementary health resources

The portal is a learning and organization tool. Families should always verify current guidance with licensed clinicians and official sources.
